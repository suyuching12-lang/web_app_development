# 專案名稱：食譜收藏夾

## 1. 專案概述
- **背景與動機**：現代家庭主婦經常在網路上看到各種食譜，但缺乏一個統整、方便收藏和查看的地方。手寫筆記容易遺失，網頁書籤或截圖又難以搜尋整理。因此，需要一個簡單易用的數位食譜收藏系統來集中管理。
- **目標用戶**：家庭主婦、喜愛居家料理的烹飪愛好者。
- **核心價值主張**：提供一個直覺、無負擔的介面，快速記錄料理名稱、所需材料與料理步驟，輕鬆打造專屬的個人食譜庫。

## 2. 功能需求
以下為系統的主要功能需求與使用者故事：

1. **新增食譜**
   - 作為 **家庭主婦**，我希望 **能新增一筆食譜（包含料理名稱、所需材料清單、料理步驟）**，以便 **記錄我新學會或想要嘗試的菜色**。
2. **瀏覽食譜列表**
   - 作為 **家庭主婦**，我希望 **可以在首頁看到我所有收藏的食譜清單**，以便 **在不知道晚餐要煮什麼時能快速尋找靈感**。
3. **查看食譜詳情**
   - 作為 **家庭主婦**，我希望 **點擊特定食譜後能看到完整的材料列表與步驟說明**，以便 **在廚房料理時可以看著螢幕一步步跟著做**。
4. **編輯食譜內容**
   - 作為 **家庭主婦**，我希望 **能隨時修改食譜的材料分量或調整料理步驟**，以便 **根據每次實際烹飪的經驗，改良出最適合家人的口味**。
5. **刪除食譜**
   - 作為 **家庭主婦**，我希望 **能刪除已經不需要或不常做的食譜**，以便 **保持食譜庫的整潔與精簡**。

## 3. 非功能需求
- **技術限制**：後端使用 Python Flask 框架，前端使用 Jinja2 模板引擎進行渲染，資料庫採用輕量級的 SQLite。
- **UI/UX 考量**：操作介面必須支援響應式設計（RWD），因為主婦們經常會在廚房裡使用手機或平板來查看食譜。字體大小需適中，避免閱讀困難。
- **效能與安全考量**：系統畫面載入應保持流暢；實作上防範基本的 SQL Injection 與 XSS 攻擊，確保使用者輸入內容的安全性。

## 4. MVP 範圍

### Must Have (必須具備)
- 基本的食譜 CRUD（建立、讀取、更新、刪除）功能。
- 食譜資料結構需包含：料理名稱、材料清單、料理步驟。

### Should Have (應該具備)
- 食譜關鍵字搜尋：能透過料理名稱來尋找食譜。
- 建立日期紀錄與最新建立排序。

### Nice to Have (有了更好)
- 圖片上傳：為食譜新增成品照片，增加豐富度。
- 分類標籤：如湯品、主菜、甜點等分類，幫助快速篩選。
- 我的最愛標記：可以把常煮的食譜釘選或標記為最愛。
